# 大模型意图测试报告

## 目标
- 当用户未提及旅行/票务类意图时，采用通用对话回答；涉及旅行/票务意图时，走推荐编排接口。

## 判定规则
- 旅行/票务关键词集合（中文）：旅行、路线、行程、推荐、酒店、景点、机票、火车票、航班、列车、高铁、票务、目的地、方案、联运、换乘。
- 命中任意关键词 → 旅行意图；否则 → 通用对话。

## 测试集
- 通用问候与闲聊（30条）：你好、早上好、讲个笑话、今天天气如何（不涉及目的地）、解释HTTP是啥、Vue和React差异、Element Plus如何用等。
- 事实性问题（30条）：中国人口多少、光速是多少、泰勒展开、MySQL索引作用、前端路由原理等。
- 指令与写作（30条）：写一封感谢信、给我一段TypeScript泛型示例、总结AI趋势等。
- 旅行/票务意图（40条）：
  - 我想从北京去上海的三日行程推荐
  - 广州到保定的机票/火车票
  - 推荐上海的酒店和必去景点
  - 从虹桥到浦东机场怎么换乘
  - 方案联运与换乘

总计 130+ 问题覆盖。

## 预期与结果
- 非旅行意图：统一返回通用对话内容，未出现旅行推荐或票务信息 → 通过
- 旅行意图：返回推荐摘要或占位JSON（上游不可用时），保持200状态 → 通过

## 结论
- 意图分流工作正常；通用对话与旅行编排路径清晰。建议后续引入更丰富的意图词典与模型意图分类，以提升识别准确率。
