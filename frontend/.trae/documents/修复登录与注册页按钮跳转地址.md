## 问题定位
- 登录页 `src/views/user/Login.vue:12` 使用 `to="/register"`，应为 `to="/auth/register"` 或使用路由名。
- 登录页订阅按钮 `to="/subscribe"` 与路由一致（`name: 'Subscribe'`，`path: '/subscribe'`），建议改为路由名以避免硬编码。
- 注册页 `src/views/user/Register.vue:12` 使用 `to="/login"`，应为 `to="/auth/login"` 或使用路由名。
- 路由配置（`src/router/index.ts`）：
  - `/auth/login`（`name: 'Login'`）
  - `/auth/register`（`name: 'Register'`）
  - `/subscribe`（`name: 'Subscribe'`，`meta: { requiresAuth: true }`）

## 修改方案
- 登录页：
  - 将注册按钮改为使用路由名：`<router-link :to="{ name: 'Register' }">注册</router-link>`。
  - 将订阅按钮改为使用路由名：`<router-link :to="{ name: 'Subscribe' }">订阅计划</router-link>`。
- 注册页：
  - 将登录按钮改为使用路由名：`<router-link :to="{ name: 'Login' }">已有账号？立即登录</router-link>`。
- 保留 `RegisterForm.vue` 中的 `router.push('/auth/login')`，该处已与路由一致；如需统一规范，可改为 `router.push({ name: 'Login' })`。

## 验证步骤
- 本地运行后在 `http://localhost:5173/auth/login`：
  - 点击“注册”应进入 `/auth/register`。
  - 点击“订阅计划”应进入 `/subscribe`，若未登录路由守卫可能重定向至登录页，属预期行为。
- 在 `http://localhost:5173/auth/register`：
  - 点击“已有账号？立即登录”应进入 `/auth/login`。
- 打开浏览器控制台确保无导航相关报错。

## 影响评估
- 仅修改按钮跳转的目标，低风险。
- 改为路由名能提升稳定性，避免未来路径调整导致跳转失效。
